# GrafanaPlexMonitor
GrafanaPlexMonitoring is an advanced Grafana dashboard designed for monitoring a multi‑user Plex environment using **Tautulli** metrics exposed through **Prometheus**.

This dashboard is intended for Plex servers hosting **4K remux libraries**, supporting **multiple remote users**, and operating under **limited WAN upload bandwidth** (for example ~350 Mbps).

The dashboard helps operators quickly identify:

- Active streaming sessions
- Transcodes vs direct play
- Users consuming the most bandwidth
- Client devices causing transcoding
- 4K playback load
- Estimated WAN saturation risk

The goal is to provide a **Network Operations Center (NOC) style view of Plex activity** so administrators can diagnose playback failures or buffering events quickly.

---

# Requirements

You must already have the following running:

- Grafana
- Prometheus
- Tautulli
- The Tautulli Prometheus exporter included in this repository

This dashboard assumes Prometheus is scraping metrics such as:

```
plex_stream_sessions
plex_stream_active
plex_stream_bitrate
```

These metrics are generated by the **Tautulli session exporter** which queries the Tautulli API.

# Installation

## 1. Install the Tautulli Prometheus Exporter

Create a directory:

```
mkdir plex-exporter
cd plex-exporter
```

Place the following files in this directory:

```
Dockerfile
tautulli_session_exporter.py
docker-compose-exporter.yml
```

Start the exporter:

```
docker compose -f docker-compose-exporter.yml up -d --build tautulli-exporter
```

The exporter will expose metrics at:

```
http://SERVER_IP:9594/metrics
```

---

## 2. Configure Prometheus

Add the exporter to your `prometheus.yml`:

```yaml
- job_name: plex_sessions
  static_configs:
    - targets: ['SERVER_IP:9594']
```

Restart Prometheus after updating the configuration.

You should now see Plex metrics when querying Prometheus.

---

## 3. Import the Grafana Dashboard

1. Open Grafana
2. Navigate to **Dashboards → Import**
3. Upload:

```
GrafanaPlexMonitoring.json
```

4. Select your **Prometheus datasource**

The dashboard will now populate with Plex session metrics.

# Dashboard Overview

GrafanaPlexMonitoring provides several operational views of your Plex server.

## Core Monitoring Panels

**Active Streams**  
Displays the total number of active Plex sessions.

**Streams by Decision**  
Breaks down playback method:

- Direct Play
- Direct Stream
- Transcode

This helps identify when client devices cannot play media natively.

**Bandwidth by User**  
Shows the bitrate being consumed by each user.

Useful for identifying bandwidth-heavy streams.

**Streams by Client Device**  
Displays which devices are currently connected (Roku, iPhone, browser, etc).

---

## Capacity and Network Monitoring

**Remote Bandwidth Consumption**  
Estimates WAN bandwidth usage from remote streams.

**WAN Saturation Prediction**  
Calculates the estimated percentage of your available upload bandwidth currently in use.

Configured for environments with ~350 Mbps upload capacity.

---

## Media Library Insights

**4K vs 1080p Session Count**  
Tracks the number of high-resolution streams currently active.

Important for servers hosting 4K remux content.

**4K Remux Risk Score**  
An operational metric combining:

- number of 4K streams
- number of active transcodes

Used as a quick signal that the server may approach resource limits.

---

## Client Troubleshooting

**Clients Forcing Transcodes**  
Identifies client devices that cannot direct play content.

This helps determine which platforms are causing server load.

---

## User Impact Panels

**Top 5 Bandwidth Users**  
Highlights which users are consuming the most bandwidth.

**Per‑User Bitrate Pressure**  
Shows bitrate usage broken down by user and client device.

# Recommended Plex Settings

For environments similar to this dashboard's design assumptions:

| Setting | Recommendation |
|-------|-------|
Internet Upload Speed | Set to your actual WAN upload speed |
Remote Stream Bitrate | Limit to ~20‑25 Mbps |
Hardware Transcoding | Enable QuickSync or GPU acceleration |

These settings help maintain stable playback under multi-user load.

---

# Notes

This dashboard visualizes **declared stream bitrate**, not burst buffering behavior used internally by Plex.

During playback startup Plex may temporarily exceed the displayed bitrate while filling client buffers.

Therefore:

- WAN saturation warnings should be treated as **early indicators**
- Buffering events may occur before graphs show full saturation
